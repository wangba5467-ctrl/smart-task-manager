### 2. 测试 Skill

**在本地测试：**

```bash

cp -r smart-task-manager ~/.openclaw/skills/

# 2. 运行安装脚本
cd ~/.openclaw/skills/smart-task-manager/bin
node setup.js

# 3. 重启网关
openclaw gateway restart

# 4. 测试功能
/new

帮我写个函数
```

**预期结果：**
- ✅ 安装脚本成功执行
- ✅ 配置文件自动创建
- ✅ Skill 正常工作
- ✅ 记忆功能正常

### 3. 版本号检查

```json

  "openclaw": {

  }
```

## 📦 发布到 GitHub

### 步骤 1：创建 GitHub 仓库

1. **登录 GitHub**
   - 访问 https://github.com
   - 登录你的账号

2. **创建新仓库**
   - 点击右上角 "+"
   - 选择 "New repository"
   - 仓库名：`smart-task-manager`
   - 描述：`智能任务管理器 - 让 OpenClaw 真正记住你的每个要求`
   - 可见性：✅ Public
   - 初始化：❌ 不添加 README、.gitignore 等

3. **创建仓库后，GitHub 会显示命令：**

```bash
echo "# smart-task-manager" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/your-username/smart-task-manager.git
git push -u origin main
```

### 步骤 2：推送代码

```bash
# 进入项目目录
cd smart-task-manager

# 初始化 Git（如果还没有）
git init

# 添加所有文件
git add .

# 提交
git commit -m "feat: 智能任务管理器 v1.0

- 智能任务路由
- 自动记忆管理
- 快速通道机制
- 流程执行引擎
- 违规检查机制"

# 添加远程仓库
git remote add origin https://github.com/your-username/smart-task-manager.git

# 推送到 GitHub
git push -u origin main
```

### 步骤 3：创建 Release

1. **在 GitHub 上：**
   - 进入仓库
   - 点击右侧 "Releases"
   - 点击 "Create a new release"

2. **填写信息：**
   - Tag：`v1.0.0`
   - Release title：`智能任务管理器 v1.0`
   - Description：复制 `CHANGELOG.md` 内容
   - 勾选：✅ Set as the latest release

3. **点击 "Publish release"**

## 📊 版本管理

### 语义化版本

**格式：`MAJOR.MINOR.PATCH`**

- **MAJOR（主版本）**：不兼容的 API 变更
  - 示例：`1.0.0` → `2.0.0`

- **MINOR（次版本）**：向后兼容的功能新增
  - 示例：`1.0.0` → `1.1.0`

- **PATCH（补丁版本）**：向后兼容的 bug 修复
  - 示例：`1.0.0` → `1.0.1`

### 发布新版本

**步骤：**

1. **更新版本号**
   ```bash
   # 更新 package.json
   npm version patch  # 或 minor, major
   ```

2. **更新 CHANGELOG**
   ```markdown
   ## [1.0.1] - 2026-03-20

   ### Added
   - 新功能 1

   ### Fixed
   - 修复 bug 1

   ### Changed
   - 变更 1
   ```

3. **提交并打标签**
   ```bash
   git add .
   git commit -m "chore: release v1.0.1"

   git tag v1.0.1
   git push origin main --tags
   ```

4. **创建 GitHub Release**
   - 在 GitHub 创建新的 Release
   - Tag：`v1.0.1`

5. **发布到 npm**
   ```bash
   npm publish
   ```
