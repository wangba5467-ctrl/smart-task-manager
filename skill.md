# 智能任务管理器 (Smart Task Manager)

**自动化任务执行和记忆管理 - 让 OpenClaw 真正记住你的每个要求**

---

## 🎯 核心功能

### 1. 智能任务路由
- 自动判断任务类型（快速通道 vs 标准流程）
- 查找并执行对应流程文档
- 支持自定义流程扩展

### 2. 自动记忆管理
- 自动记住你的每个要求
- 自动检索和应用记忆
- 支持 QMD 和 Memos

### 3. 流程执行引擎
- 严格按流程执行
- 不跳步骤、不省确认
- 违规自我检查和纠正

## 📋 使用方法

### 触发方式

**自动触发（推荐）：**
```
任何任务都会自动触发本 skill
```

**显式触发：**
```
执行任务：[任务描述]
帮我：[任务描述]
开始：[任务描述]
```

### 快速通道任务

**简单任务自动快速执行：**

```
# 信息查询
帮我找 XXX 文件
Python 版本是多少？
看一下 XXX 文档

# 计算类
1024 * 1024 = ?
1MB 等于多少 KB？
```

**响应时间：秒级（2-10 秒）**

**复杂任务走完整流程：**

```
# 公众号写作
帮我写篇文章

帮我开发登录功能

# 数据分析（需要先创建流程）
分析这个数据
```

**执行时间：按流程执行，质量保证**

## 🧠 记忆管理

### 自动记忆写入

**触发条件：**
- ✅ 用户说"记住"
- ✅ 用户表达要求
- ✅ 用户表达偏好
- ✅ 用户纠正错误

**示例：**
```
你：记住：代码必须加注释

AI：✅ 已记住：用户要求"代码必须加注释"
```

### 自动记忆检索

**每次对话开始时：**
- QMD 自动检索相关记忆
- 显示相关记忆（如果有）
- AI 主动应用记忆

**示例：**
```
你：帮我写个函数

AI：我记得你要求"代码必须加注释"

[自动应用，代码带详细注释]
```

## ⚙️ 配置文件

### 首次使用自动设置

**Skill 会自动创建以下配置文件：**

1. **任务类型映射表.md**
   - 定义任务类型到流程的映射
   - 位置：`~/.openclaw/workspace/任务类型映射表.md`

2. **快速通道任务列表.md**
   - 定义快速通道任务
   - 位置：`~/.openclaw/workspace/快速通道任务列表.md`

3. **通用任务执行流程.md**
   - 通用任务流程（兜底）
   - 位置：`~/.openclaw/workspace/通用任务执行流程.md`

4. **流程文档模板.md**
   - 创建新流程的模板
   - 位置：`~/.openclaw/workspace/流程文档模板.md`

### 自定义配置

**编辑任务类型映射表：**
```markdown
| **新任务** | 触发词 | 新任务流程.md | ✅ 已有 |
```

**编辑快速通道列表：**
```markdown
| **新任务类型** | 说明 | 示例 |
```

## 📚 创建自定义流程

### 步骤 1：使用模板

```bash
# 复制模板
cp ~/.openclaw/workspace/流程文档模板.md \
   ~/.openclaw/workspace/新任务流程.md
```

### 步骤 2：填写内容

按照模板结构填写：
- 任务概述
- 流程步骤
- 工具资源
- 注意事项

### 步骤 3：更新映射表

在 `任务类型映射表.md` 中添加：

```markdown
| **新任务** | 触发词 | 新任务流程.md | ✅ 已有 |
```

### 步骤 4：测试

```
你：帮我[执行新任务]

AI：[自动使用新流程]
```

## 🎓 工作原理

### 任务执行流程

```
用户下达任务
    ↓
⚡ 快速通道检查
    ├─ 是 → 🚀 快速执行（秒级）
    └─ 否 → 📋 标准流程
            ↓
        QMD 记忆检索
            ↓
        任务类型识别
            ↓
        查找流程文档
            ├─ 找到 → 📖 读取并执行
            └─ 没找到 → 🔧 通用流程
```

### 违规检查机制

**执行过程中自我检查：**
- [ ] 是否读了流程文档？
- [ ] 是否跳过了步骤？
- [ ] 是否提供了选项？

**如果违规：**
```
⚠️ 违规！我应该先读流程文档
[停止]
[读取流程文档]
[重新执行]
```

## 🛠️ 故障排查

### 问题 1：Skill 不触发

**检查：**
```bash
# 查看技能列表
openclaw skill list

# 检查 skill.json
cat ~/.openclaw/skills/smart-task-manager/skill.json
```

**解决：**
- 确保 skill.json 存在
- 重启网关：`openclaw gateway restart`

### 问题 2：找不到配置文件

**检查：**
```bash
ls ~/.openclaw/workspace/
```

**解决：**
- Skill 会自动创建配置文件
- 或手动运行：`openclaw skill install smart-task-manager --setup`

### 问题 3：记忆不工作

**检查：**
```bash
# 检查 QMD 配置
cat ~/.openclaw/openclaw.json | grep memory
```

**解决：**
- 确保 `"memory.backend": "qmd"`
- 确保 `"memory.citations": "auto"`
